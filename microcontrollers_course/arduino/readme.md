# Знакомство с микроконтроллером ATmega
Здесь представлен код, который был загружен в микроконтроллер `ATmega328p` для реализации взаимодействия нескольких устройств по Wi-Fi с помощью протокола `TCP`. Код соответствует `Arduino`, работающей в роли `клиента`.
## Оборудование
 - Микроконтролллер `ATmega328p`, клон `Arduino`
 - Wi-Fi модуль `ESP8266-01`

## Логика работы
Данный проект направлен на знакомство с контролерром `ATmega` и реализацию взаимодействия нескольких микроконтроллеров. В качестве данных для обмена передавались команды на включение либо выключение (в случае данной платы выключение) светодиода на Arduino, работающей в режиме сервера. Рассматриваемая плата Arduino отправляет каждые `3` секунды команду на отключение светодиода на другую Arduino, выполняющую функцию сервера
### Реализация протокола UART 
Для работы с Wi-Fi модулем необходим протокол `UART`, но в Arduino только один аппаратный протокол, который используется для обмена данными с компьютером, к которому он подключен. Поэтому необходимо реализовать программный `UART`. В этом поможет использование библиотеки `SoftwareSerial`. Будут использованы пины `10` и `11`.
### Особенности TCP на Arduino
В данном протоле есть четкое разделение на роли: `клиент` и `сервер`. `Arduino` не может одновременно быть как клиентом, так и сервером. Возможен только один режим работы.
